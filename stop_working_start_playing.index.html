Game Theory — "ตรรกะเบื้องหลังทุกการตัดสินใจในออฟฟิศ"

โลกของการทำงานจริงไม่ได้ขับเคลื่อนด้วยความใจดีหรือโชคชะตาเพียงอย่างเดียว แต่มันขับเคลื่อนด้วยสิ่งที่นักเศรษฐศาสตร์รางวัลโนเบลเรียกว่า "ทฤษฎีเกม (Game Theory)" — ศาสตร์ที่อธิบายว่าทำไมคนเราถึงตัดสินใจแบบนั้น และจะเดาทางคนอื่นได้อย่างไร

ในออฟฟิศ ทุกคนรอบตัวคุณ ตั้งแต่ประธานบริษัทไปจนถึงพนักงานชั่วคราว ต่างก็เป็น "ผู้เล่น" (Players) ที่มีเป้าหมายและทางเลือกของตัวเอง ทุกคำพูดที่คุณเอ่ย ทุกอีเมลที่ส่ง และทุกการตัดสินใจที่คุณเลือก คือการวางหมากบนกระดานที่มีผลตอบแทน (Payoffs) และความเสี่ยงรออยู่เสมอ

กลยุทธ์แก้เกม — Game Analysis & Strategy

เปลี่ยน Zero-sum Game เป็น Cooperative Game

🎯 วิเคราะห์ตัวเกม

สถานการณ์นี้คือ "เกมแบบผลรวมเป็นศูนย์" (Zero-sum Game) ในโลกของพี่ดาว เธอเชื่อว่าความเก่งมีจำกัด ถ้าน้องนนท์ได้คะแนนเต็ม 10 คะแนนของเธอจะเหลือแค่ 0 ทันที การกั๊กข้อมูลคือวิธีที่ทำให้เธอมีอำนาจเหนือกว่า เพราะในออฟฟิศ "ข้อมูลคือแต้มต่อ" (Information Asymmetry)

1

เปลี่ยนเกมด้วย Incentive Compatibility: ทำให้เป้าหมายของน้องนนท์ไปรวมกับเป้าหมายของพี่ดาว — เมื่อความสำเร็จของนนท์กลายเป็นผลงานของดาวด้วย พี่ดาวจะเปลี่ยนจากการกั๊กข้อมูลมาเป็นการสนับสนุน

2

สร้าง Cooperative Game: วิธีแก้ Zero-sum Game ที่ดีที่สุดคือการเปลี่ยนให้เป็น "เกมความร่วมมือ" — มนุษย์จะเปลี่ยนพฤติกรรมก็ต่อเมื่อ "ผลตอบแทนใหม่ (Payoff) มันคุ้มค่ากว่าเดิม"

กลยุทธ์แก้เกม — Mechanism Design

เปลี่ยนกติกาให้ IT รับผิดชอบ Productivity ด้วย

🎯 วิเคราะห์ตัวเกม

ต้องใช้กลยุทธ์ "การออกแบบกลไก" (Mechanism Design) คือการเปลี่ยนกติกาให้ IT ต้องรับผิดชอบต่อ "ความลื่นไหลในการทำงาน" ของแผนกอื่นด้วย ไม่ใช่แค่ความปลอดภัยเพียงอย่างเดียว เพื่อบีบให้ IT ต้องไปหาเทคโนโลยีที่ "ทั้งปลอดภัยและใช้ง่าย" มาแทน

1

เพิ่ม Productivity KPI ให้ IT: วัดผลฝ่าย IT ทั้ง Security Score และ User Satisfaction Score เพื่อบีบให้ต้องหาเทคโนโลยีที่ตอบโจทย์ทั้งสองด้าน

2

ลงทุนใน Smart Tech: Biometric Login, Single Sign-On (SSO) หรือ Passwordless Authentication ช่วยแก้ Trade-off ได้โดยไม่ต้องเลือกข้างใดข้างหนึ่ง

Mechanism Design — รางวัลโนเบลที่เปลี่ยนวิธีบริหารคน

มนุษย์มีสัญชาตญาณในการหาทางที่ "เหนื่อยน้อยที่สุดแต่ได้ผลตอบแทนมากที่สุด" เสมอ

ในออฟฟิศ คำพูดมักจะมีราคาถูก (Cheap Talk) ใครจะพูดให้ตัวเองดูดียังไงก็ได้ แต่คนจะเชื่อถือคุณก็ต่อเมื่อคุณส่ง "สัญญาณที่มีต้นทุน" (Costly Signaling) ออกไปเท่านั้น

บารมีไม่ได้เกิดจากการบอกว่าเราเก่ง แต่มันเกิดจากการที่คนอื่น "ตีความ" จากสิ่งที่เรายอมแลกเพื่อให้ได้งานนั้นมา — หากคุณส่งสัญญาณผิด ชีวิตการเป็น Manager ของคุณจะเหนื่อยเป็นสองเท่า เพราะต้องคอยพิสูจน์ตัวเองซ้ำแล้วซ้ำเล่า

นี่คือความขัดแย้งระหว่าง "ผลประโยชน์ระยะสั้น" vs "ความยั่งยืนระยะยาว" (Hyperbolic Discounting) — ผู้นำต้องใช้กลยุทธ์ Backward Induction คือเอาภาพความพินาศในอนาคตมาแสดงให้บอร์ดเห็นเป็นตัวเลข เพื่อลด Payoff ของการลดคนให้เหลือศูนย์ และเพิ่มค่าให้กับการรักษาคนแทน

กลยุทธ์แก้เกม — Backward Induction

เอาอนาคตมาเปลี่ยนการตัดสินใจวันนี้

🔍 วิเคราะห์ตัวเกม

นี่คือความขัดแย้งระหว่าง "ผลประโยชน์ระยะสั้น" กับ "ความยั่งยืนระยะยาว" (Hyperbolic Discounting) — มนุษย์มักเลือกความสุขตรงหน้ามากกว่าความสำเร็จในอนาคต ผู้นำที่ติดกับดักนี้มักจะทิ้ง "ระเบิดเวลา" ไว้ให้คนข้างหลัง

1

ใช้ Backward Induction: แสดงให้บอร์ดเห็นว่า "ต้นทุนของการไม่เปลี่ยนในวันนี้" จะมหาศาลแค่ไหนในอนาคต — เช่น ค่าจ้างพนักงานใหม่ที่สูงกว่า ความเสียหายจากความรู้ที่สูญหาย และต้นทุนชื่อเสียงในตลาดแรงงาน

2

Finite loss เพื่อ Infinite win: ผู้นำที่ยิ่งใหญ่กล้ายอมรับความพ่ายแพ้ในเกมสั้น (ตัวเลขไตรมาสนี้ไม่สวย) เพื่อรักษาชัยชนะในเกมยาว (ทีมที่แข็งแกร่งในอีก 5 ปีข้างหน้า)
